# معمارية النظام — محاسب مشروعي

> وثيقة مرجعية لطبقات التطبيق، التخزين، المزامنة، والأمان.

---

## 1️⃣ الطبقة الأمامية (Frontend)

| العنصر | التفاصيل |
|--------|----------|
| **التقنيات** | React + Vite + PWA |
| **الدور** | واجهة المستخدم، حسابات محلية، تخزين مؤقت |

### قاعدة مهمة

- **`localStorage` = cache فقط** — ليس المصدر الأساسي للبيانات.
- المصدر الأساسي: Supabase (PostgreSQL) مع نسخ محلي للمزامنة والعمل دون اتصال.

---

## 2️⃣ طبقة المزامنة (Sync Layer) — القلب النابض

هذه الطبقة هي التي تميّز **SaaS ناجح** عن تطبيق عادي.

### التدفق (Offline First Sync)

```
إجراء المستخدم (مثلاً: إضافة فاتورة)
        ↓
    حفظ محلي فوري (UX سريع)
        ↓
    إدخال في طابور المزامنة (Queue)
        ↓
    إرسال إلى السحابة في الخلفية
```

- المستخدم يرى النتيجة فوراً دون انتظار السيرفر.
- عند انقطاع الإنترنت: الطابور يتراكم، والمزامنة تتم عند عودة الاتصال.
- لا فقدان بيانات؛ تجربة سلسة على الموبايل والشبكات الضعيفة.

---

## 3️⃣ الطبقة الخلفية (Backend) — Supabase

| المكوّن | الاستخدام |
|---------|-----------|
| **PostgreSQL** | قاعدة البيانات الرئيسية |
| **Auth** | تسجيل الدخول والمستخدمين |
| **Realtime** | تحديثات فورية (اختياري) بين الأجهزة |
| **Storage** | ملفات (تقارير، مرفقات) |
| **Edge Functions** | منطق خادم (إشعارات، تكاملات، معالجة دفع) |

---

## 4️⃣ تصميم قاعدة البيانات (Multi-tenant)

كل البيانات مرتبطة بـ **مساحة عمل (Workspace)** حتى لا يرى تاجر بيانات تاجر آخر.

### جداول أساسية (كل جدول يحتوي `workspace_id`)

- `users`
- `workspaces`
- `workspace_members`
- `products`
- `transactions`
- `invoices`
- `expenses`
- `customers`
- `orders`

---

## 5️⃣ طبقة الأمان — Row Level Security (RLS)

- استخدام **RLS** في Supabase على كل الجداول.
- السياسة: المستخدم يرى فقط الصفوف التابعة لـ `workspace_id` الذي هو عضو فيه.
- لا يمكن الوصول لبيانات تاجر آخر من نفس الـ API.

---

## 6️⃣ استراتيجية الأداء

| الآلية | الهدف |
|--------|--------|
| **Pagination** | 20 عنصر لكل صفحة في الفواتير، الحركات، المنتجات |
| **Lazy Loading** | تحميل الصفحات الثقيلة عند الدخول فقط: التقارير، النسخ الاحتياطي، الاستيراد (عبر `React.lazy`) |
| **Code Splitting** | تحميل الصفحات والمكتبات عند الحاجة (موجود في المشروع) |

---

## ملخص

- **Frontend:** React + Vite + PWA، و`localStorage` للكاش فقط.
- **Sync:** Offline First — حفظ محلي فوري ثم طابور مزامنة إلى السحابة.
- **Backend:** Supabase (DB, Auth, Realtime, Storage, Edge Functions).
- **DB:** تصميم multi-tenant بـ `workspace_id` في كل جدول.
- **Security:** RLS حتى يرى كل مستخدم بياناته فقط.
- **Performance:** Pagination + Lazy Loading + Code Splitting.

هذا الهيكل يدعم تطبيقاً جاهزاً للنمو كـ SaaS مع تجربة مستخدم قوية (سريع، يعمل دون اتصال، آمن).

# تحليل المنافسين والميزة التنافسية — تطبيق محاسب (Mahaseb)

## 1. العميل المستهدف

**محاسب** موجّه إلى:

- **أصحاب المشاريع الصغيرة** — محلات، ورش، حرفيون، مقدمي خدمات يحتاجون تتبع مبيعات ومصروفات وإيرادات.
- **أصحاب المشاريع الأونلاين** — من يعملون من المنزل أو عبر الإنترنت ويريدون حسابات بسيطة وواضحة.
- **التجارة الإلكترونية** — بائعون على منصات أو متاجر إلكترونية يريدون إدارة مخزون ومبيعات وعملاء وفواتير دون تعقيد ERP.

الواجهة بالعربية، والمنتج يجمع بين **البساطة** و**شمول المحاسبة اليومية** لهذه الفئات.

---

## 2. وصف التطبيق (من الكود الحالي)

التطبيق **محاسب** هو تطبيق ويب/موبايل (PWA + Android عبر Capacitor) لإدارة الحسابات والمبيعات والمصروفات للعميل المستهدف أعلاه.

### الميزات الرئيسية المستخرجة من المشروع

| المجال | الميزات |
|--------|---------|
| **المالية** | إيرادات ومصروفات، تصنيفات، تقارير، تقرير يومي، أهداف مبيعات شهرية |
| **المبيعات والمخزون** | منتجات، مبيعات (أوردرات) مع عميل وكمية وسعر، حالة (مكتمل/قيد الانتظار) |
| **العملاء والفواتير** | عملاء، فواتير (مع تواريخ استحقاق ومدفوع/غير مدفوع)، صفحة فواتير العملاء، صفحة مشتريات وديون الموردين |
| **الدين والذمم** | تتبع ديون العملاء، ديون الموردين والمشتريات |
| **التقارير والتصدير** | تقارير، تقرير يومي، تصدير Excel، إنشاء PDF (فواتير، حزمة محاسب)، إرسال تقارير جاهزة عبر واتساب |
| **الاستيراد** | استيراد حركات (إيرادات/مصروفات) أو مبيعات من ملف Excel مع تعيين الأعمدة |
| **النسخ الاحتياطي** | نسخ احتياطي واستعادة البيانات |
| **الإشعارات** | مركز إشعارات داخل التطبيق |
| **الاشتراكات** | صفحة اشتراكات (ربط بدفع ناجح) |
| **العلامة التجارية** | إعدادات للعلامة التجارية (مثل اسم الشركة) |
| **واتساب** | تقارير ملخص وتقرير يومي ونص جاهز للفواتير للإرسال عبر واتساب |
| **تقني** | PWA، عمل دون اتصال (مع cache)، مزامنة سحابية (Supabase أو Firebase)، دعم أندرويد (Capacitor) |

## 3. تحليل المنافسين

### 3.1 منافسون في نفس الفئة (محاسبة/إدارة لمشاريع صغيرة ومحلات)

- **تطبيقات محاسبة عامة عربية (سحابية أو هجينة)**  
  عادة تتضمن: فواتير، مصروفات/إيرادات، عملاء، تقارير، أحياناً مخزون. كثير منها إما ثقيل (ERP) أو بسيط جداً (فقط فواتير). تطبيقك يجمع بين البساطة وشمول المحاسبة اليومية + المبيعات + الفواتير + التقارير.

- **حلول نقاط البيع (POS)**  
  تركيز على الكاشير والباركود والورديات. تطبيقك لا يركز على POS بالدرجة الأولى بل على **الحسابات والمبيعات والتقارير** مع إمكانية استخدامه من الموبايل في المحل.

- **برامج محاسبة صيدليات (فينيكس، فارماكير، فاتورة، بلاجن، وازن)**  
  متخصصة في الصيدليات: تأمين صحي، صلاحية أدوية، باركود، معتمدة من الزكاة والضريبة. تطبيقك **عام** (محلات، مشاريع، ويمكن استخدامه في صيدلية بسيطة) وليس بديلاً متخصصاً لهذه الحلول في السوق السعودي المعتمد ضريبياً.

### 3.2 مقارنة سريعة (اتجاهات السوق)

| المعيار | تطبيق محاسب (هذا المشروع) | منافسون نموذجيون (حلول صيدليات/محاسبة) |
|--------|---------------------------|----------------------------------------|
| **الجمهور المستهدف** | أصحاب المشاريع الصغيرة، المشاريع الأونلاين، والتجارة الإلكترونية | صيدليات، أو شركات متوسطة، أو متاجر مع POS |
| **التخصص** | عام (مرن) | كثير منها متخصص (صيدليات، ضرائب سعودية) |
| **اللغة والواجهة** | عربية كاملة | عربية غالباً |
| **المزامنة السحابية** | نعم (Supabase/PostgreSQL أو Firebase) مع cache محلي | غالباً سحابي أو سيرفر محلي |
| **عمل دون اتصال** | نعم (PWA + cache) | يختلف حسب المنتج |
| **تطبيق موبايل أصلي** | أندرويد (Capacitor) + PWA | غالباً ويب أو تطبيقات مخصصة |
| **واتساب (تقارير جاهزة)** | نعم (ملخص، يومي، فاتورة) | أقل شيوعاً كمنتج جاهز |
| **الاستيراد من Excel** | نعم (حركات + مبيعات) | يختلف |
| **الاعتماد الضريبي (مثلاً السعودية)** | غير واضح من الكود | كثير من منافسي الصيدليات معتمد |

## 4. الميزة التنافسية (Competitive Advantages)

بناءً على الكود والهيكل الحالي، يمكن تلخيص **أهم المزايا التنافسية** كما يلي:

### 4.1 مزايا واضحة

1. **تجربة عربية كاملة + بساطة الاستخدام**  
   واجهة موحدة للمحاسبة والمبيعات والفواتير والتقارير دون تعقيد ERP، مناسبة لأصحاب المشاريع الصغيرة والمشاريع الأونلاين والتجارة الإلكترونية.

2. **عمل دون اتصال + مزامنة سحابية**  
   استخدام localStorage كـ cache مع Supabase (أو Firebase) يعطي سرعة في الواجهة ويسمح بالعمل عند انقطاع النت ثم المزامنة لاحقاً — مفيد للمشاريع الصغيرة والتجارة الإلكترونية التي تعمل من أكثر من جهاز.

3. **التكامل مع واتساب (تقارير جاهزة)**  
   نصوص جاهزة للملخص والتقرير اليومي والفاتورة للإرسال عبر واتساب توفر وقت صاحب المشروع وتجربة مألوفة (واتساب هو القناة الفعلية للتواصل مع العملاء في كثير من الأسواق).

4. **استيراد من Excel**  
   استيراد الحركات والمبيعات من Excel يقلل الحواجز أمام الانتقال من جداول أو أنظمة قديمة إلى التطبيق.

5. **تقارير وتصدير**  
   تقارير + تقرير يومي + تصدير Excel + PDF (فواتير وحزمة محاسب) تغطي احتياجات المحاسبة اليومية والعرض على محاسب خارجي أو جهة دعم.

6. **PWA + أندرويد**  
   عدم الاعتماد على متجر تطبيقات فقط (PWA) مع إمكانية توزيع أندرويد عبر Capacitor يوسع الوصول ويقلل احتكاك التثبيت.

### 4.2 نقاط يمكن تعزيزها لزيادة التميز

- **الاعتماد الضريبي (إن وُجدت جهة معتمدة في الدولة المستهدفة)**  
  إذا أضيفت فواتير متوافقة مع متطلبات الزكاة والضريبة (مثلاً السعودية)، يزيد التميز في السوق المحلي.

- **الباركود ونقاط البيع (POS)**  
  إضافة مسح باركود وواجهة كاشير بسيطة تجعل التطبيق أقرب لـ "نظام مبيعات + محاسبة" في محل فعلي.

- **ربط مع منصات التجارة الإلكترونية (اختياري)**  
  تكامل مع متاجر أو قنوات بيع أونلاين يزيد القيمة لعملاء التجارة الإلكترونية.

---

## 5. ملخص

- **العميل المستهدف:** أصحاب المشاريع الصغيرة، أصحاب المشاريع الأونلاين، والتجارة الإلكترونية.
- **التطبيق:** محاسبة ومبيعات وفواتير وعملاء وتقارير وتصدير واستيراد Excel ونسخ احتياطي، وتقارير واتساب، وPWA وأندرويد، ومزامنة سحابية مع عمل دون اتصال.
- **المنافسون:** يتراوحون بين حلول محاسبة عامة، نقاط بيع، وبرامج متخصصة. محاسب يخدم **المشاريع الصغيرة والأونلاين والتجارة الإلكترونية** بدون تعقيد ERP.
- **الميزة التنافسية الأبرز:**  
  الجمع بين **العربية + البساطة + العمل دون اتصال مع المزامنة + تقارير واتساب الجاهزة + استيراد Excel** في منتج واحد يخدم العميل المستهدف مباشرة.

يمكن استخدام هذا المستند كأساس لـ pitch أو لوثيقة استراتيجية منتج أو لتحديد أولويات التطوير (مثل الضريبة أو الباركود) لتعزيز الميزة التنافسية في سوق معين.

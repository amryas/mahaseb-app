# ربط المتاجر وشركة الشحن

## ملخص

- **ربط المتاجر:** الطلبات من الموقع (API)، Shopify، WooCommerce، و Easy Order تُجلب إلى التطبيق وتظهر كـ **مبيعات** و**إيراد** تلقائياً.
- **حالة التوصيل:** كل طلب قادم من المتجر له حالة توصيل: `قيد التوصيل` ← يمكن تحديثها إلى `تم التسليم` أو `مرتجع` يدوياً أو لاحقاً عبر API شركة الشحن.
- **شركة الشحن:** حالياً التحديث **يدوي** من صفحة «ربط المتاجر» أو من صفحة «المبيعات». ربط API (بوستا، أرامكس، إلخ) مخطط له لاحقاً.

---

## ماذا تم تنفيذه

### 1) ربط المتاجر — الطلبات والمبيعات أوتوماتيك

- **ربط الموقع (API):** إدخال رابط API الطلبات (وربما مفتاح API). زر «جلب الطلبات والمبيعات» أو مزامنة تلقائية عند فتح الصفحة.
- **Shopify:** إدخال رابط المتجر + Access Token. اختبار الاتصال ثم جلب الطلبات.
- **WooCommerce:** إدخال رابط الموقع + Consumer Key + Consumer Secret. اختبار ثم جلب الطلبات.
- **Easy Order:** جلب طلب برقم المعرف وإضافته كإيراد ومبيعة.

كل طلب مُجلب يُحفظ كمبيعة مع:
- `source`: `'website' | 'shopify' | 'woocommerce' | 'easy_order'`
- `externalOrderId`: معرف الطلب في المنصة
- `deliveryStatus`: `'pending'` (قيد التوصيل) حتى يتم تحديثه

### 2) مزامنة تلقائية

- في صفحة «ربط المتاجر» خيار: **«جلب الطلبات والمبيعات تلقائياً عند فتح هذه الصفحة»**.
- عند تفعيله، عند فتح الصفحة يُجلب تلقائياً من الموقع و Shopify و WooCommerce (حسب ما هو مُعدّ) ويُضاف الجديد فقط إلى المبيعات والإيراد.

### 3) شركة الشحن — التحديث اليدوي

- قسم **«شركة الشحن — الطلبات المسلّمة والمرتجعة»** في نفس الصفحة.
- يعرض كل الطلبات القادمة من المتاجر ذات الحالة `قيد التوصيل`.
- لكل طلب: زر **«تم التسليم»** (تحديث الحالة فقط) أو **«مرتجع»** (إلغاء الإيراد وتسجيل مرتجع، وإرجاع المخزون إن وُجد ربط منتج).

نفس التحديث (تم التسليم / مرتجع) متاح من **صفحة المبيعات** لأي مبيعة قادمة من المتجر.

### 4) التقارير والداشبورد

- المبيعات ذات الحالة `ملغى` أو `مرتجع` (بما فيها الطلبات المُحدّثة كمرتجع من الشحن) لا تدخل في إجمالي المبيعات ولا الإيراد، كما هو معتاد في التطبيق.

---

## إعداد WooCommerce

1. من لوحة ووردبريس: **WooCommerce → الإعدادات → متقدم → REST API**.
2. إنشاء مفتاح API (قراءة/كتابة أو قراءة حسب الحاجة).
3. نسخ **Consumer key** و **Consumer secret** وإدخالهما في التطبيق مع رابط الموقع (مثل `https://mysite.com`).

---

## ربط API شركة الشحن (مستقبلاً)

- يمكن لاحقاً إضافة تكامل مع بوستا أو أرامكس أو غيرها:
  - جلب حالة الشحنات (تم التسليم / مرتجع) عبر API.
  - مطابقة الطلبات في التطبيق (مثلاً برقم الطلب أو رقم الـ waybill إن أضفناه).
  - تحديث `deliveryStatus` و `status` (مرتجع) تلقائياً.

الحقلان `shippingProvider` و `shippingApiKey` موجودان في الإعدادات استعداداً لهذا الربط.

# دليل التحديثات والخطوات — محاسب مشروعي

**راجع هذا الملف لمعرفة ماذا تفعل ولقائمة الاقتراحات المستقبلية.**

---

## آخر التحديثات (جاهز لآخر مرحلة)

- **تسجيل الحسابات:** شاشة إنشاء الحساب واختيار الحساب مُحدَّثة وواضحة (اسم الحساب/الشركة فقط، بدون تعقيد).
- **داشبورد احترافية:** لوحة تحكم بتصميم أوضح — عنوان ووصف، بطاقة «بيع من واتساب»، ثم الإحصائيات والرسوم البيانية.
- **بيع من واتساب:** من لوحة التحكم يمكنك تسجيل مبيعة وردت من واتساب (المبلغ، اسم العميل، رقم واتساب). يتم إضافتها كإيراد ومبيعة، مع إمكانية فتح واتساب لإرسال تأكيد للعميل. رقم واتساب للتواصل من الإعدادات.
- **جاهز لـ SaaS:** إعدادات الاشتراك في البيانات (subscriptionStatus, subscriptionPlan)، وملف `src/config/saas.js` لربط سيرفر لاحقاً عبر `VITE_SAAS_API_URL`.
- **الكود:** إزالة ما لا يلزم، معالجة أخطاء واضحة، صلاحيات المدير/المحاسب/الموظف مضبوطة.

---

## ما تم تنفيذه سابقاً (شاشة العملاء)

- **الاسم، رقم الهاتف، العنوان:** في صفحة «العملاء والفواتير» → تبويب «العملاء» يمكنك تسجيل الاسم ورقم الهاتف والعنوان وحفظهم.
- **تصدير إلى Excel:** زر «تصدير العملاء إلى Excel» يحمّل ملفاً واحداً (مثل `عملاء_2025-03-06.xlsx`) يحتوي على كل العملاء بالأعمدة: الاسم، رقم الهاتف، العنوان.

## ماذا تفعل الآن (خطوة بخطوة)

### 1) تشغيل التطبيق للتطوير
```bash
cd c:\projects\app
npm install
npm run dev
```
ثم افتح المتصفح على `http://localhost:5173`.

### 2) بناء نسخة ويب للإنتاج
```bash
npm run build
```

### 3) بناء APK لأندرويد
```bash
npm run build:apk
npx cap open android
```
من Android Studio: **Build** → **Build Bundle(s) / APK(s)** → **Build APK(s)**.

### 4) تفعيل Firebase
انسخ `.env.example` إلى `.env` واملأ قيم Firebase (راجع `FIREBASE_SETUP.md`).

## اقتراحات قوية لتطوير التطبيق (تأثير حقيقي على العمل)

هذه أفكار تفرّق التطبيق وتزيد الطلب عليه وترضي مصلحة الضرائب والبنوك والعملاء:

---

### محاسبة واحتراف

| الاقتراح | لماذا قوي؟ |
|----------|-------------|
| **تصدير جاهز للضرائب / الإقرار** | تصدير الحركات والفواتير بصيغة مناسبة للمحاسب أو للضرائب (تواريخ، فئات، أرقام) — مطلوب جداً لأي مشروع رسمي. |
| **حزمة شهرية للمحاسب** | زر «تصدير شهر للمحاسب»: PDF أو Excel فيه كل الحركات والفواتير والمبيعات والمصروفات مرتبة — العميل يرسلها لمحاسبه مرة في الشهر. |
| **تقرير تدفق نقدي (Cash Flow)** | تنبيه: «الشهر الجاي عندك التزامات X والمتوقع تدخل Y، الرصيد المتوقع Z» — يمنع مفاجآت نقص السيولة. |
| **مقارنة أشهر / سنوات** | رسم بياني: هذا الشهر vs الشهر اللي فات، أو هذه السنة vs اللي فاتت (مبيعات، مصروفات، ربح) — قرارات مبنية على أرقام. |

---

### مبيعات ومخزون

| الاقتراح | لماذا قوي؟ |
|----------|-------------|
| **وضع كاشير سريع (Point of Sale)** | شاشة مبسطة للموبايل: اختيار منتج + كمية + إجمالي + إرسال واتساب أو طباعة إيصال — منافسة لبرامج الكاشير الغالية. |
| **تنبيه مخزون + اقتراح طلب** | لما الصنف يوصل تحت «حد التنبيه» يطلع تنبيه واضح، واختياري: «اقتراح طلب من المورد X بكمية Y». |
| **أفضل منتجات ربحية / أفضل عملاء** | تقرير: أي المنتجات تربحك أكثر، وأي العملاء يشتري أكثر أو يدين أكثر — تركيز على الأهم. |
| **ربط طابعة حرارية أو بلوتوث** | طباعة إيصال من الجوال مباشرة (للشحنات أو المحلات) — يقلل الأخطاء ويرضي العميل. |

---

### عملاء وعلاقات

| الاقتراح | لماذا قوي؟ |
|----------|-------------|
| **ربط العميل بالفواتير والمبيعات** | عند الفاتورة أو المبيعة: اختيار العميل من القائمة → يظهر الهاتف والعنوان تلقائياً للإرسال أو الشحن. |
| **تقارير حسب العميل** | لكل عميل: إجمالي مشتريات، إجمالي مديونية، آخر تاريخ شراء — مفيد للمتابعة والتحصيل. |
| **تنبيه ديون قديمة** | «عندك 3 عملاء مدينين من أكثر من 30 يوم» — يذكّر بالتحصيل قبل ما يتراكم. |

---

### تقني وتجربة

| الاقتراح | لماذا قوي؟ |
|----------|-------------|
| **تعدد مستخدمين وصلاحيات** | أدوار: مدير (كل الصلاحيات)، محاسب (حركات وتقارير)، كاشير (مبيعات فقط) — مناسب لو المحل فيه أكثر من شخص. |
| **نسخ احتياطي تلقائي (سحابي)** | نسخ أسبوعي أو شهري على Firebase أو Google Drive — حماية من ضياع البيانات. |
| **وضع عدم اتصال قوي** | كل الوظائف الأساسية تعمل بدون نت، والمزامنة تلقائياً عند رجوع النت — مهم للجوال والمحلات. |
| **إشعارات فعلية (Push)** | إشعار على الموبايل: طلب جديد من Easy Order، فاتورة استحقت، صنف وصل حد التنبيه — تفاعل أسرع. |

---

### توسع وربط

| الاقتراح | لماذا قوي؟ |
|----------|-------------|
| **ربط واتساب Business API (إن أمكن)** | إرسال فاتورة أو تقرير تلقائياً من التطبيق للعميل على واتساب بدون نسخ ولصق — تجربة احترافية. |
| **ربط منصات شحن** | استيراد شحنات (أو تكلفة شحن) من منصة شحن وإضافتها كمصروف أو ربطها بالطلب — توفير وقت. |
| **فواتير إلكترونية مع QR** | العميل يمسح QR على الفاتورة ويرى التفاصيل وحالة الدفع — ثقة وشفافية. |
